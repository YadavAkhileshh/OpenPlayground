## ğŸ› ï¸ Technical Implementation

### Map Generation Logic
The dungeon is generated by simulating a "drunk walker" that moves randomly across a grid. Every time it moves into a "Wall" tile, it converts it into a "Floor" tile. This continues until 40% of the map is carved out, ensuring organic, cavern-like structures.

[Image of a flowchart showing the steps of the Drunkard's Walk algorithm for procedural generation]

### Rendering Engine
Instead of DOM manipulation (which is slow for games), this project uses a single `<canvas>` element.
- **Layer 1:** Background and Floor/Wall tiles.
- **Layer 2:** Interactive elements (Stairs).
- **Layer 3:** Entities (Player and Goblins).
- **Layer 4:** UI Overlay (HTML/CSS).

### Enemy AI
The current AI utilizes a Manhattan Distance calculation to track the player. If the player enters the "aggro radius," the enemy moves towards the player's coordinates using a simple axis-alignment logic.
