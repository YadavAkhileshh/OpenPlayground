Data Structures

Array

A collection of elements identified by index or key. Elements are stored in contiguous memory locations.

Time Complexity

Operation Best Average Worst Space
Access O(1) O(1) O(1) O(1)
Search O(1) O(n) O(n) O(1)
Insertion O(1) O(n) O(n) O(1)
Deletion O(1) O(n) O(n) O(1)

Common Operations

// Initialize array
int arr[5] = {1, 2, 3, 4, 5};

// Access element
int x = arr[2]; // O(1)

// Traverse array
for(int i = 0; i < 5; i++) {
  cout << arr[i] << " ";
}

Linked List

A linear data structure where elements are stored in nodes. Each node contains a data field and a reference (link) to the next node.

Time Complexity

Operation Best Average Worst Space
Access O(1) O(n) O(n) O(n)
Search O(1) O(n) O(n) O(1)
Insertion (head) O(1) O(1) O(1) O(1)
Deletion (head) O(1) O(1) O(1) O(1)

Types of Linked Lists

  • Singly Linked List: Each node points to the next node
  • Doubly Linked List: Each node points to both next and previous nodes
  • Circular Linked List: Last node points back to the first node

Stack

A linear data structure that follows the LIFO (Last In First Out) principle. Elements are added and removed from the same end (top).

Time Complexity

Operation Complexity
Push (Insert) O(1)
Pop (Delete) O(1)
Peek (Top) O(1)
Search O(n)

Common Applications

  • Function call management (call stack)
  • Expression evaluation and syntax parsing
  • Undo mechanisms in text editors
  • Backtracking algorithms

Tree

A hierarchical data structure consisting of nodes connected by edges. Each tree has a root node and zero or more child nodes.

Time Complexity (Binary Search Tree)

Operation Best Average Worst
Access O(log n) O(log n) O(n)
Search O(log n) O(log n) O(n)
Insertion O(log n) O(log n) O(n)
Deletion O(log n) O(log n) O(n)

Common Tree Types

  • Binary Tree: Each node has at most 2 children
  • Binary Search Tree: Left child < Parent < Right child
  • AVL Tree: Self-balancing BST
  • Heap: Complete binary tree with heap property

Algorithms

Sorting Algorithms

Algorithms that arrange elements of a list in a particular order (ascending or descending).

Time & Space Complexity

Algorithm Best Average Worst Space Stable
Bubble Sort O(n) O(n²) O(n²) O(1) Yes
Insertion Sort O(n) O(n²) O(n²) O(1) Yes
Selection Sort O(n²) O(n²) O(n²) O(1) No
Merge Sort O(n log n) O(n log n) O(n log n) O(n) Yes
Quick Sort O(n log n) O(n log n) O(n²) O(log n) No
Heap Sort O(n log n) O(n log n) O(n log n) O(1) No

When to Use Which?

  • Quick Sort: General purpose, fastest average case
  • Merge Sort: Stable, good for linked lists
  • Heap Sort: Guaranteed O(n log n), minimal memory
  • Insertion Sort: Small datasets, nearly sorted data

Searching Algorithms

Algorithms to find an element within a data structure.

Time Complexity

Algorithm Best Average Worst Space Data Structure
Linear Search O(1) O(n) O(n) O(1) Array, Linked List
Binary Search O(1) O(log n) O(log n) O(1) Sorted Array
Depth-First Search O(|V| + |E|) O(|V| + |E|) O(|V| + |E|) O(|V|) Graph, Tree
Breadth-First Search O(|V| + |E|) O(|V| + |E|) O(|V| + |E|) O(|V|) Graph, Tree
// Binary Search Implementation
int binarySearch(int arr[], int l, int r, int x) {
  while (l <= r) {
    int mid = l + (r - l) / 2;
    if (arr[mid] == x) return mid;
    if (arr[mid] < x) l = mid + 1;
    else r = mid - 1;
  }
  return -1;
}

Complexity Comparison

Big O Notation Cheat Sheet

Below is a comparison of common time complexities from fastest to slowest:

Complexity Name Example For n=100
O(1) Constant Array access, Hash table lookup 1 operation
O(log n) Logarithmic Binary search, Balanced tree operations ~7 operations
O(n) Linear Linear search, Traversing an array 100 operations
O(n log n) Linearithmic Merge sort, Heap sort, Quick sort (average) ~664 operations
O(n²) Quadratic Bubble sort, Selection sort, Nested loops 10,000 operations
O(2ⁿ) Exponential Fibonacci (naive), Subset generation 1.27×10³⁰ operations
O(n!) Factorial Traveling salesman (brute force) 9.33×10¹⁵⁷ operations

Space Complexity Guidelines

  • O(1): Algorithm uses a fixed amount of space regardless of input size
  • O(n): Space grows linearly with input size (arrays, stacks)
  • O(n²): Space grows quadratically (2D arrays, adjacency matrices)
