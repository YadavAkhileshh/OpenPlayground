How to Implement

  1. Add thumbnail to your projects.json file
    { "title": "My Awesome Project", "category": "utility", "description": "A cool project...", "icon": "ri-tools-line", "coverStyle": "background: linear-gradient(...)", "thumbnail": "./preview.png" // ‚Üê Add this line! }
  2. Add your image file

    Place your thumbnail image (preview.png, screenshot.jpg, etc.) in your project folder

  3. That's it!

    The system automatically detects and displays your thumbnail with smooth animations and fallback support

Before vs After

Before (Icon Only)

  • All cards look similar
  • Harder to distinguish projects
  • Less visual appeal
  • Slower project scanning

After (With Thumbnails)

  • Unique visual identity
  • Easy project recognition
  • Enhanced visual hierarchy
  • Faster browsing experience

Key Features

Optional Field

Add thumbnails only when you want. Existing projects work without changes.

Graceful Fallback

Missing images? No problem. System falls back to gradient + icon.

Responsive

Looks great on all devices with automatic sizing and scaling.

Smooth Animations

Hover effects and transitions for a polished user experience.

Error Handling

Broken links? System shows fallback icon automatically.

Both Views

Works in both grid view and list view seamlessly.
